Artifact 2

Machine Learning vs. Deep Learning

This artifact is a written analysis comparing one traditional machine learning example and one deep learning example from the course lesson. It was revised for portfolio presentation so that the content is easier for a professional audience to follow while still demonstrating technical understanding.

Project Context

This report was created as part of my coursework to show that I can distinguish between machine learning and deep learning, connect them to real-world use cases, and explain why one approach may be more appropriate than another depending on the problem.

Objective

The goal of this artifact was to evaluate two examples from the lesson—email spam detection as a machine learning example and image recognition as a deep learning example—and explain why each approach is suitable for its specific application.

Process

I reviewed the lesson content on machine learning and deep learning, selected one example from each category, and then connected each example to a real-world application. After that, I compared the strengths of both approaches, focusing on factors such as feature extraction, data complexity, computational needs, and practical fit for the task.

Tools & Technologies Used
Microsoft Word APA 7 Formatting Machine Learning Concepts Deep Learning Concepts Course Research
Key Examples Covered

Machine Learning: Email spam detection using structured, predefined features such as word frequency, sender information, and suspicious links.

Deep Learning: Image recognition using convolutional neural networks to learn patterns directly from raw image data.

Why This Artifact Matters

This artifact reflects my ability to translate technical theory into practical understanding. It shows that I can compare AI methods critically, explain them clearly, and connect academic learning to real-world systems such as email filtering and facial recognition.

Value Proposition

For employers and collaborators, this work demonstrates that I do more than memorize AI terms. I can evaluate when a method is appropriate, explain trade-offs, and communicate technical ideas in a way that supports smarter business and technology decisions.
